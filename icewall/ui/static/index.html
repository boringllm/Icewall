Build from CVEs — fetches each fix commit from GitHub, then distills it
How this works

Each CVE id is looked up on OSV, its GitHub fix commit(s) fetched (api.github.com + raw.githubusercontent.com), the changed functions extracted, and each distilled into knowledge.

Needs network access to GitHub. A token (set via config) raises the rate limit. Best for a handful of specific CVEs — for bulk, use the Import from dataset tab.

Seed from skills populates the base from Icewall's bundled per-CWE skills with no network — a good first run.

Import from a dataset — bulk-build from a downloaded corpus
Point at the dataset folder — Icewall finds the .sql.gz, converts it to SQLite (once), then imports.
Which dataset?

CVEfixes — a downloaded SQLite corpus that bundles the vulnerable & patched code, so it needs no GitHub. Point Icewall at the dataset folder; it converts the .sql.gz to SQLite once, then imports.

OSV — per-ecosystem dumps that only reference fix commits, so this path fetches code from GitHub.

Uses the distiller & embedding from the Endpoints tab. The limit and Injection CWEs only filter keep cost bounded.
